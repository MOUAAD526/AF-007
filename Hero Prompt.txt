## THE EXACT PROCESS PER RECORD — 3 PHASES

### PHASE 1 — SLOT 1 (one focused session, full depth)

Give this slot your complete attention. Do not think about Slot 2 yet.

Run the full 7-step process:

**STEP 1 — TITLE SCAN**
Read the post_title. Scan for trigger words from the style trio table (Part 12).
Record: TRIGGER FOUND: [word] OR NO TRIGGER FOUND.

**STEP 2 — STYLE SELECTION**

Slot 1 anchor: use `dashboard.recommended_style_number` as the starting point.

Apply the Slot 1 Diversity Rule before confirming:
- Check the style ledger.
- If the recommended style has already appeared as Slot 1 in 2 of the last 10 records,
  rotate to the next compatible unused style from the Part 12 trio table.
- State: SLOT 1: Style N [DASHBOARD ANCHOR] or SLOT 1: Style N [DIVERSITY OVERRIDE — reason].

Slot 2 selection:
- Open the Part 12 trio table. Find the style family that contains the finalized Slot 1 style.
- Select the Slot 2 companion style from that family.
- Apply the Slot 2 Diversity Rule:
  Slot 2 must never match Slot 1 for the same record.
  No style may appear as Slot 2 more than 3 times across the full run
  unless all other options in the family are exhausted.
- State: SLOT 2: Style N [reason].

Update the style ledger immediately after Step 2.

**STEP 3 — COMPRESS THE TITLE**
Run the Part 3 compression process.
Build one 3-line banner headline:
- Total: 6–10 words across all lines
- Line 3 (amber punch): 2–4 words, never more than 5
- This headline is shared across both slots — compress it once, use it in both.
State the result before writing any prompts.

**STEP 4 — HERO COPY SPEC**
Confirm all 5 copy elements per Part 4:
- Eyebrow: 3–5 words, ALL-CAPS
- Headline: the 3-line compressed version, final line FDA050 amber
- Subheading/dek: 1 complete sentence, 12–22 words, Inter Regular
- Visual type: PHOTO (emotional/life-situation post) or GRAPHIC (system/data/comparison post)
- Dominant anchor: the one thing the eye hits first — state it in one line.
  If you cannot name it in one line, simplify the concept before proceeding.

**STEP 5 — PASS THE HERO SIMPLICITY TEST (Part 5D)**
Answer all 6 questions. Resolve any NO before writing the prompt.

**STEP 6 — WRITE THE SLOT 1 PROMPT**
Use the Mode B generator-safe template (Part 5B — THE GEMINI TEMPLATE block).
Source the composition language from the MODE B TRANSLATION block of the selected
style in Part 2. Do not translate the Mode A spec in real time.
Write for the eye. No px values. No hex codes in the composition body.
No coordinate language. No CSS terms.
All color references must be plain-language descriptions only:
amber orange, deep navy, warm cream, brand blue, etc.

After the main composition, append all 3 of the following blocks in this order:

TYPOGRAPHY HIERARCHY BLOCK (required in every prompt):
TYPOGRAPHY HIERARCHY:
The headline must be large and confident but must not dominate the entire layout.
The headline may occupy no more than 45–50% of the usable text area.
The subheading/dek must be clearly readable at full website hero display size.
Chips, labels, bottom-bar text, and supporting captions must remain legible
at thumbnail preview size. Use a professional editorial web-design ratio between
headline and supporting text — not a billboard or poster ratio. No text element
should visually crush another element. The image must have clear hierarchy,
breathing room, and usable contrast between headline, dek, labels, and data details.

text

CREATIVE DIRECTION BLOCK (required in every prompt):
CREATIVE DIRECTION:
The rules above define the structure, content, and brand compliance.
Within those constraints, apply your own editorial design judgment for the remaining
visual decisions — breathing room, spatial rhythm, subtle texture, atmosphere, and
any micro-detail not explicitly described. The goal is a finished image that feels
like it was designed by a senior editorial art director, not rendered from a
specification. Prioritize feeling natural and attractive over being technically precise.

text

ANTI-ANNOTATION EXPORT BLOCK (required in every prompt):
CRITICAL EXPORT RULES:
Render the final image only. No CSS annotation text, no pixel labels,
no hex code overlays, no coordinate markers, no design spec text of any kind.
Every element in this image is a visual design element — nothing is a label
or instruction. If any part of this description sounds like a build instruction,
render the visual result it describes, not the instruction itself.

text

Approved slots are locked — never touch them again.

## THE HARD CHECKLIST (10 points — 1 point each)

Score each prompt out of 10. Pass = 8.5 or higher.
CHECK 1 — STYLE MATCH
Slot 1 uses dashboard.recommended_style_number (or a valid diversity override).
Slot 2 uses the correct companion from the Part 12 trio table.
PASS = correct style selected and reason is documented.
FAIL = style ignored, guessed, or overridden without a valid reason.

CHECK 2 — DOMINANT ANCHOR
There is exactly one dominant anchor stated explicitly before the composition.
It is describable in one line.
PASS = one clear focal point named before the layout description.
FAIL = two competing anchors, vague anchor, or no anchor stated.

CHECK 3 — HEADLINE WORD COUNT
Total words across all headline lines: 6–10.
Line 3 (amber): 2–4 words, never more than 5.
PASS = word count confirmed within range.
FAIL = under 6 total, over 10 total, or Line 3 exceeds 5 words.

CHECK 4 — EYEBROW SPEC
3–5 words, ALL-CAPS.
PASS = confirmed.
FAIL = too short, too long, not in caps, or written as a sentence.

CHECK 5 — DEK SPEC
1 complete sentence. 12–22 words. Not a fragment. Not two sentences.
PASS = confirmed.
FAIL = fragment, double sentence, under 12 words, or over 22 words.

CHECK 6 — MODE B COMPLIANCE
Zero px values anywhere in the composition body.
Zero hex codes anywhere in the composition body.
Zero coordinate language (x160, y400, etc.).
Zero CSS terms (border-radius, rgba, letter-spacing, opacity values).
Zero font name + size specifications inline in the layout.
Color references are ALL plain-language descriptions only.
PASS = none of the above appear in the composition body.
FAIL = any one of the above is present.

CHECK 7 — BANNED ELEMENTS
None of the following appear anywhere in the prompt:
URLs or domain names, CSS spec text, pixel labels, cartoon illustrations,
stock-finance clichés (piggy banks, coins, suited professionals pointing at graphs,
happy people holding credit cards), 3D rendering, beveled effects,
colors outside the official AF palette, shame framing, panic visuals.
PASS = none present.
FAIL = any one present.

CHECK 8 — TEASE RULE (Part 5 — The Tease Rule)
The banner creates desire to read the article.
It does NOT teach the full system or summarize the post.
After 3 seconds, the reader feels seen and curious — not informed and satisfied.
PASS = banner teases, does not explain.
FAIL = banner lists steps, shows full methodology, or answers the question completely.

CHECK 9 — REQUIRED BLOCKS PRESENT
All 3 required blocks appear at the end of the prompt in correct order:

TYPOGRAPHY HIERARCHY

CREATIVE DIRECTION

CRITICAL EXPORT RULES
PASS = all 3 present, in order.
FAIL = any block missing or out of order.

CHECK 10 — PSYCHOLOGY GUARDRAILS
No shame framing (still broke, can't afford, you're doing it wrong).
No panic visuals without implied agency or a next step.
No diagnosis-as-hook framing for ADHD, trauma, anxiety, avoidance topics.
For comparison prompts: no winner chip, no checkmark badge, no Best Pick label.
PASS = all guardrails respected.
FAIL = any violation present.

text
